# EasyEDA Agent Workflow Example

This is a catalog-level example. The final EasyEDA-specific workflow should live in `oaslananka/easyeda-mcp-pro` after the product plugin and skills are added there.

## Goal

Use an AI agent with EasyEDA MCP support to assist with project navigation, component search, validation, and design documentation.

## Expected flow

1. Identify the EasyEDA project context.
2. Inspect available project metadata.
3. Confirm the target workflow: review, component search, validation, or documentation.
4. Use supported MCP tools to gather project information.
5. Produce a concise engineering summary.
6. Flag risks, missing data, and validation gaps.
